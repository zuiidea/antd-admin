# 国际化（i18n）使用说明

本项目通过 Umi 的 `plugin-locale`（基于 `react-intl`）管理多语言，翻译资源位于 `src/locales`（例如 `en-US.json`、`zh-CN.json`）。下面说明如何：新增语言、增加翻译条目、切换语言与关闭多语言。示例以新增日语 `ja-JP` 为例。

## 1. 新增一种语言（长期支持）

1. 在 `src/locales/` 下创建新的翻译文件，文件名为语言标识，例如：`src/locales/ja-JP.json`。

2. 文件内容为 JSON 格式的翻译对象（可嵌套）：

```json
{
    "welcome": "ようこそ",
    "login": {
        "username": "ユーザー名",
        "password": "パスワード",
        "submit": "ログイン"
    }
}
```

3. 若希望 Ant Design 组件也使用对应的本地化（日期/组件文案），在 `src/app.tsx` 的 `pick` 函数中添加对应的 AntD locale 引入与映射，例如：

```ts
import jaJP from 'antd/locale/ja_JP';
// ...
if (l === 'ja-JP') return jaJP;
```

4. 在语言选择列表（例如 `src/components/TopbarControls.tsx` 的 `localeMenuItems`）中加入新语言项：

```ts
{ key: 'ja-JP', label: '日本語' }
```

5. 重启开发服务器：

```bash
pnpm run dev
```

Umi 会在构建时将 `src/locales` 下的文件编入 `localeInfo`，页面即可使用新语言。

提示：如不想重启或想动态注册翻译，可使用运行时 API `addLocale(name, messages, extraLocales)`（见下一节）。

## 2. 增加一条多语言文本（翻译条目）

1. 在所有语言的 JSON 文件中添加相同的 id（推荐同时维护 `en-US.json` 与 `zh-CN.json`，新语言也应添加）：

`src/locales/en-US.json`

```json
"my.feature.greeting": "Hello from feature"
```

`src/locales/zh-CN.json`

```json
"my.feature.greeting": "来自功能的问候"
```

2. 在组件中使用 `useIntl()` 获取 `intl`，并调用 `formatMessage`：

```ts
import { useIntl } from 'umi';
const intl = useIntl();
const text = intl.formatMessage({ id: 'my.feature.greeting', defaultMessage: 'Hello' });
```

建议使用 `useIntl()`（或 `injectIntl`）以保证语言切换时组件可自动刷新并显示新文本。

## 3. 运行时切换或注册语言

- 切换语言（会写入 `localStorage` 并触发刷新或事件）：

```ts
import { setLocale } from 'umi';
// 切换并刷新页面
setLocale('ja-JP', true);
// 切换但不刷新（触发事件更新）
setLocale('zh-CN', false);
```

- 运行时注册（无需重启）：

```ts
import { addLocale, setLocale } from 'umi';
import jaMessages from '@/locales/ja-JP.json';
import jaAntd from 'antd/locale/ja_JP';

addLocale('ja-JP', jaMessages, { antd: jaAntd, momentLocale: 'ja' });
setLocale('ja-JP', false);
```

`addLocale` 会把翻译合并到内部 `localeInfo`，并可同时提供 AntD 与（可选的）moment 本地化配置。

## 4. 关闭多语言（禁用 i18n）

若想永久关闭多语言功能：

1. 在 `src/app.tsx` 的 `LocaleWrapper` 中，直接返回固定的 AntD locale（忽略 `localStorage` 中的 `umi_locale`）：

```ts
const pick = (l: string | null) => {
    return zhCN; // 强制使用中文
};
```

2. 移除或隐藏语言切换控件（例如删除 `src/components/TopbarControls.tsx` 中的语言菜单项或 Dropdown）。

3. 可选：删除 `src/locales` 中其它语言文件，仅保留目标语言文件以减少构建体积。

如果只是临时禁止用户切换，建议仅执行第 2 步（隐藏切换），以便未来快速恢复多语言支持。

## 5. 调试与常见问题

- 新增语言后无显示：确认已重启 dev server（或使用 `addLocale` 动态注册），并在 `app.tsx` 中为该语言提供 AntD locale。  
- 翻译未生效或缺失：检查 `src/locales/*.json` 是否包含对应 id，且 `formatMessage` 中的 id 拼写正确。  
- 想自动抽取翻译 id：仓库未内置抽取脚本，建议按需集成 `lingui`、`babel-plugin-react-intl` 或其他工具并在 `package.json` 中添加相应脚本。
